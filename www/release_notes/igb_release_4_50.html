IGB 4.50 release notes.

IGB 4.50 adds new capabilities for file parsing and display manipulation.

A few new features were added in version 4.48.

Highlights:
  • Version 4.50
    • There is better support for sharing files between IGB and the UCSC Genome Browser.

      Since many users of IGB are familiar with the UCSC Genome Browser and have created files specifically for use with it, we have made some changes that allow the two programs to work together more easily.
      • Added support for reading the Wiggle Graph Format
      • Parsing of Track Lines is consistent with the UCSC browser for formats "BED", "GFF", "PSL" and "WIG". IGB makes use of these properties useScore, url, description and visibility.
      • IGB also support parsings of Track Lines for "GFF3" format, though the UCSC browser does not yet support this format. (Track line must begin with "##track".)
      • The range coordinate box in IGB now allows input of coordinates in the same format used by UCSC, e.g. "chr1:1,000,000-2,000,000", so that you may easily copy and paste coordinates from one program to the other.
      • Allows coloring exons, and other features, based on their score. (Works with "BED", "GFF" and "GFF3" files.) This can be turned on and off from a track line, or from a menu item. The colors are interpolated from the given foreground and background colors.


    • Added a small display to always show the current position of the hairline. (Something similar was possible in earlier versions with "Toggle Hairline Label", but that label could cover-up some annotations.)

    • Added support for parsing "CHP" files for expression chips. The genomic locations of probesets are not included in these files, but IGB will automatically load that information as needed from a DAS-based web server.

    • The buttons for performing arithmetic operations on two graphs have been fixed. (In version 4.48, they all performed subtraction, instead of the full four operations.)

    • The DAS/2 client now complies completely with the final DAS/2 specification for feature data retrieval. (This version of the DAS/2 specification is the final "frozen" version.)

    • Now allowing configuration of multiple web links from each feature type.

    • You can now choose to show or hide the "+" and "-" strands of data.

    • You can now load a FASTA formatted file that contains sequence data for multiple chromosomes (or other sequences). FASTA format is recommended only for small genomes.

    • When "BAR" format files contain datat from multiple genomes, the data from all those genomes will be read and made available through the "Data Access" tab.
