Creating a QuickLoad File System


It is easy to create your own QuickLoad directory for use with IGB.
You may wish to do this to hold local copies of public sequence and
annotation data or to hold your private annotation data for your
organism(s) of interest.

All that is required is a set of directories and files organized in
a particular way. You must then let IGB know where the main
directory is located. The files may be located either on a
local file system or on a web server.



Structure of the QuickLoad Directory

QuickLoad requires a collection of directories, one for each
genome, plus a few index files.

(Althogh we speak of "genomes" and "chromosomes" here, you can
generalize this to any collection of annotated sequences.)


The top-level directory must contain:
One sub-directory for each genome
File: contents.txt
File: synonyms.txt
File: das_servers.txt

Each genome directory must contain:
Required File: annots.txt
Required: Either "liftAll.lft" or "mod_chromInfo.txt"
Optional: refseq.brs
Optional: A set of files with genomic sequence for each chromosome
Optional: Any additional annotation files you wish

When setting up your own QuickLoad directory, you may wish
to refer to the Affymetrix QuickLoad server: http://netaffxdas.affymetrix.com/quickload_data/



Formats of QuickLoad files


"contents.txt"

The file "contents.txt" is a list of the genomes available in your
QuickLoad server.
The file has two columns, separated by tabs.
Column 1 contains the names of the genome directories.
Column 2 contains human-readable descriptions of the genomes.


"synonyms.txt"

The file "synonyms.txt" contains a list of synonyms for genomes
and chromosomes. Each line contains any number of synonyms for an
individual sequence, separated by tabs.

This file helps IGB recognize identical sequences that have
different names on different DAS servers.


"annots.txt"

A list of the annotation files in a given genome directory.
You should not list sequence data files here, but only list
annotation files.

The annotation files can be in a wide variety of formats.
The formats will be recognized based on the filename extensions.


"liftAll.lft"

Describes how contigs are assembled into chromosomes.
If you do not have this information available, you may use
a "mod_chromInfo.txt" file instead.

Each line contains
CONTIG_START tab CONTIG_NAME tab CONTIG_LENGTH tab CHROMOSOME_NAME tab CHROMOSOME_LENGTH


"mod_chromInfo.txt"

Lists the names of the chromosomes and their lenghts.
Each line contains
NAME tab LENGTH.

You must provide either a "mod_chromInfo.txt" or "liftAll.lft" file.



Optional Sequence and Annotation Files

Each genome directory can contain as few or as many annotation files
as you wish. Each annotation file should be listed in the
"annots.txt" file.

A wide variety of file formats is accepted, and the files are
recognized based on their filename extensions, such as ".psl".

It is not required, but your files will load much faster if you
first convert them to some binary formats designed for use in IGB.
Instructions on creating these binary files will be given later.

Each genome directory can optionally contain sequence data for some
or all of the chromosomes. Sequence files should use the binary
"bnib" format designed for use in IGB. Sequence files must have
names that match the name of the corresponding chromosome,
for example "chr12.bnib".

You may optionally include a "refseq.brs" file containing
refseq data in a ".brs" (Binary RefSeq) format.
This format will be documented elsewhere.
If this optional file is present and listed in annots.txt,
it will be read by default when you choose that genome.


Configuring IGB to use your QuickLoad files


Starting with IGB version 5.4, it is possible to have arbitrarily many QuickLoad servers. There are default Bioviz and Affymetrix QuickLoad servers, but you can add more servers. Go to File->Preferences->Data Sources and look for the "Add Server" panel. Specify your server's URL (or local directory). The URL should look similar to one of these:
  • A web site: "http://netaffxdas.affymetrix.com/quickload_data/"
  • A local Windows file: "file:///c:/Documents and Settings/name/quickload/"
  • A local Unix/Mac file: "file:///home/susan/quickload/"
Note the three slashes in "file:///".
Specify a friendly server name, and specify the "QuickLoad" type. Click "Add Server". The information for the new server will immediately be available.
