Simple Map

Now let's get busy. The GenoViz SDK comes with a number of glyphs. You can find them in the com.affymetrix.genoviz.glyph package. We will now build a small application that reads a configuration file and draws a map from the data in the file.

Here is an example of a configuration file.


The format of the configuration file is pretty straight forward. Each line starts with a keyword. One of "Range", "Offset", or "Glyph". Next come integers for the beginning and end. The glyph lines also contain a configuration string. This string is passed directly to the configure method for the glyph. This configuration string can take the keywords listed below.

-colorString a color name like "red" or "nicePaleBlue"
-glyphtypeString the name of a class implementing GlyphI
-mirrorboolean When true the glyph will negate the offset if end < beginning.
-offsetint the distance from the primary axis
-widthint the "thickness" of the glyph (orthogonal to the primary axis)

Looking at the source we see that parsing code has been added. For each line in the configuration file the appropriate method of a NeoMap is called.
